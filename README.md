# pdf2txf

Convert Solium/Shareworks (Morgan Stanley) stock plan PDF statements into TXF files for import into TurboTax Desktop.

## Requirements

- Python 3.9+

## Quick Start

The simplest way to convert a PDF to TXF:

```bash
./pdf2txf.sh solium.pdf
```

On first run, this automatically creates a virtual environment and installs dependencies. It then converts the PDF to TXF and validates the output. The TXF file is saved as `solium.txf` alongside the PDF.

Then import into TurboTax Desktop (Windows): **File > Import > From TXF File** and select `solium.txf`.

## Scripts

### pdf2txf.sh — One-step conversion (recommended)

Converts a PDF to TXF and validates the result.

```bash
./pdf2txf.sh <input.pdf> [output.txf] [payer_name]
```

| Argument | Default | Description |
|----------|---------|-------------|
| `input.pdf` | (required) | Input PDF file |
| `output.txf` | `<input>.txf` | Output TXF file |
| `payer_name` | `Morgan Stanley` | Payer/institution name |

Examples:

```bash
# Default output (solium.txf) and payer (Morgan Stanley)
./pdf2txf.sh solium.pdf

# Custom output file and payer
./pdf2txf.sh solium.pdf output.txf "Morgan Stanley Smith Barney"
```

### pdf2txf.py — PDF to CSV converter

Extracts tax data from a PDF and outputs a CSV file. This is the underlying converter called by `pdf2txf.sh`.

```bash
python3 pdf2txf.py <input.pdf> [options]
```

| Option | Description |
|--------|-------------|
| `-o`, `--output FILE` | Output file path. If omitted, prints to stdout. |
| `--payer NAME` | Payer/institution name. |
| `-v`, `--verbose` | Show extracted PDF text and parser diagnostics. |
| `--show-all` | Show unmatched dollar amounts that couldn't be mapped to a TXF code. |

Examples:

```bash
# Generate CSV
python3 pdf2txf.py solium.pdf -o solium.csv --payer "Morgan Stanley"

# Debug: see what text was extracted from the PDF
python3 pdf2txf.py solium.pdf -v --show-all
```

### txf_decoder.py — TXF file validator

Parses a TXF file, checks for format errors, and prints a human-readable summary of all records with totals.

```bash
python3 txf_decoder.py <file.txf>
```

### csv2txf.py — CSV to TXF converter (format testing)

Converts a CSV file (generated by `pdf2txf.py`) into TXF with multiple format variations. Useful if the default TXF format doesn't work with your tax software.

```bash
python3 csv2txf.py <file.csv> [--only {A,B,C,D}]
```

This generates four `.txf` files with different format variations:

| File | N Code | $ Line Order | Description |
|------|--------|-------------|-------------|
| `*_A.txf` | N717 (8949 codes) | Proceeds, Cost Basis | |
| `*_B.txf` | N717 (8949 codes) | Cost Basis, Proceeds | |
| `*_C.txf` | N717 (8949 codes) | Cost Basis, Proceeds, Wash Sale | 3 $ lines |
| `*_D.txf` | N323 (Schedule D) | Cost Basis, Proceeds | Default format used by pdf2txf.py |

## End-to-End Workflow

```
                          ./pdf2txf.sh (does all steps)
                         ┌──────────────────────────────────────────────────┐
                         │                                                  │
solium.pdf ──> pdf2txf.py ──> solium.csv ──> csv2txf.py ──> solium.txf ──> txf_decoder.py ──> validate
```

**Recommended:** Use `./pdf2txf.sh solium.pdf` which runs all steps automatically.

**Manual steps:**

```bash
# Step 1: Extract PDF to CSV
python3 pdf2txf.py solium.pdf -o solium.csv --payer "Morgan Stanley"

# Step 2: Convert CSV to TXF
python3 csv2txf.py solium.csv --only D
mv solium_D.txf solium.txf

# Step 3: Validate
python3 txf_decoder.py solium.txf

# Step 4: Import into TurboTax Desktop (Windows)
#   File > Import > From TXF File > select solium.txf
```

## TXF Format Details

The generated TXF uses classic Schedule D codes (N321/N323) with CRLF line endings, which is the format confirmed to work with TurboTax Desktop on Windows.

Example record:

```
TD
N323
C1
L1
PACME Corp
D01/15/2023
D06/30/2025
$5000.00
$3000.00
^
```

| Line | Meaning |
|------|---------|
| `TD` | Transaction detail record |
| `N323` | Schedule D long-term capital gain/loss |
| `C1` | Copy 1 |
| `L1` | Line 1 |
| `PACME Corp` | Security name |
| `D01/15/2023` | Date acquired |
| `D06/30/2025` | Date sold |
| `$5000.00` | Cost basis |
| `$3000.00` | Proceeds |
| `^` | End of record |

## Supported PDF Formats

| Source | Auto-detected by |
|--------|-----------------|
| Solium/Shareworks (Morgan Stanley) stock plan 1099-B | "STOCK PLAN ACCOUNT", "Morgan Stanley Capital Management" |
| Generic brokerage 1099-B | "1099-B", "Proceeds From Broker" |
| 1099-INT | "Interest Income", "1099-INT" |
| 1099-DIV | "Ordinary Dividends", "1099-DIV" |

## Adding a Custom Parser

When your PDF uses a layout that the built-in parsers don't handle, you can add a custom one. The Solium parser (`SoliumParser` in `parsers.py`) is a good reference implementation.

### Step 1: Inspect the extracted text

```bash
python3 pdf2txf.py your-doc.pdf -v
```

Note the exact text layout — spacing, labels, and how amounts appear.

### Step 2: Create a parser class in `parsers.py`

```python
class MyBrokerParser:
    """Parser for MyBroker's specific PDF format."""

    @staticmethod
    def detect(text: str) -> bool:
        # Return True if this PDF is from MyBroker
        return "MyBroker Financial" in text

    @classmethod
    def parse(cls, text: str) -> list[TaxRecord]:
        records = []
        # Example: MyBroker lists trades as:
        # "SOLD: AAPL | Qty: 100 | Proceeds: $15,000.00 | Basis: $12,000.00"
        pattern = re.compile(
            r"SOLD:\s+(?P<security>\S+)\s+\|.*?"
            r"Proceeds:\s+\$(?P<proceeds>[\d,]+\.\d{2})\s+\|.*?"
            r"Basis:\s+\$(?P<basis>[\d,]+\.\d{2})"
        )
        for m in pattern.finditer(text):
            records.append(TaxRecord(
                txf_code=323,  # long-term (use 321 for short-term)
                description="Long-term capital gain",
                amount=parse_amount(m.group("proceeds")) or 0.0,
                cost_basis=parse_amount(m.group("basis")),
                security_name=m.group("security"),
                form_type="1099-B",
            ))
        return records
```

### Step 3: Register it in `pdf2txf.py`

Add your parser to the `parse_pdf` function:

```python
from parsers import MyBrokerParser

# In parse_pdf(), add before the generic fallback:
if MyBrokerParser.detect(text):
    if verbose:
        print("Detected MyBroker content")
    records.extend(MyBrokerParser.parse(text))
```

### Step 4: Test

```bash
python3 pdf2txf.py mybroker-statement.pdf -v -o test.csv
python3 csv2txf.py test.csv --only D
python3 txf_decoder.py test_D.txf
```

## Troubleshooting

### TurboTax shows no transactions after import

1. Make sure you're using **TurboTax Desktop** (not Online — Online does not support TXF import).
2. Run `python3 txf_decoder.py solium.txf` and check for errors.
3. If the default format doesn't work, generate format variations with `csv2txf.py` and test each one.

### No recognized tax records found

1. Run with `-v` to see extracted text. If empty, the PDF is likely image-based (scanned) — this tool requires text-based PDFs.
2. Run with `--show-all` to see dollar amounts found but not mapped to TXF codes.
3. Your PDF may use a layout not yet supported. See "Adding a Custom Parser" above.

### Verifying totals

Compare the decoder's totals against your PDF statement's summary page to ensure all transactions were captured.
